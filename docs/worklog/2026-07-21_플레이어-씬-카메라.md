### 8) 씬 네비게이션 UI — SceneLoader 확장 + 마우스 클릭 UI
> "기존 SceneManager을 확장시켜서 간단한 UI를 만들고 해당 UI를 마우스로 누르면 Title에서 스테이지로 이동, 스테이지 초기화(콘솔에 초기화라고 출력), 다음 스테이지로 이동, 타이틀로 돌아가기 등을 구현해."

- `Assets/Scripts/Managers/SceneLoader.cs` (수정) — `ResetStage()` 추가
  - 콘솔에 정확히 `"초기화"` 출력 → `GameManager.Instance?.ResetStageState()` → Stage 씬 재로드
  - 기존 `LoadTitle / LoadStage(int) / LoadNextStage`는 그대로 재사용(중복 배관 X)
- `Assets/Scripts/UI/SceneNavigatorUI.cs` (신규) — 현재 씬에 따라 버튼 구성
  - Title: `[스테이지 시작]` / Stage: `[스테이지 초기화] [다음 스테이지] [타이틀로 돌아가기]` / 기타(Shop·Result): `[다음 스테이지] [타이틀로 돌아가기]`
  - `DontDestroyOnLoad` + static 인스턴스 가드(씬 재로드 시 중복 방지), `sceneLoaded` 구독으로 씬 전환마다 버튼 재구성
- Title 씬에 `SceneNavigatorUI` GameObject 배치(컴포넌트 부착) → 저장

> "uGUI Canvas + Button 방식으로 다시 만들어줘 그리고 docs 문서 읽고 이번 작업내용과 프롬프트 입력내용 정리해."

- 최초엔 IMGUI(`OnGUI` + `GUI.Button`)로 구현했다가, 요청에 따라 **uGUI 방식으로 전면 재작성**.
  - 런타임에 `Canvas`(Screen Space Overlay, `CanvasScaler` 1920×1080) + `VerticalLayoutGroup` 패널 + `Button`(Image·Text 자식)을 코드로 생성 → 씬 UI 수작업 세팅 불필요.
  - `EventSystem` 없으면 자동 생성. 프로젝트 입력 설정이 **레거시 Input Manager**(`activeInputHandler: 0`)라 `StandaloneInputModule` 사용.
  - 버튼 라벨 폰트: 내장 `LegacyRuntime.ttf`. 마우스 오버/프레스 색 피드백 지정.
- 검증: `assets-refresh` 후 Console 컴파일 에러 0건. (플레이 모드 진입/게임뷰 캡처 툴은 이 세션에 미노출 → 실제 화면 캡처 검증은 담당자가 Play로 확인 필요.)

### 9) 입력 분리 — 방향키=카메라 / WASD=플레이어 이동
> "다시, 커밋하고 방향키로는 기존 방식대로 플레이어가 고정된 채 카메라가 움직이고, wasd로는 플레이어 구체가 상하좌우로 움직이도록 해. 스피드는 일단 3.0 정도로 하고 후에 수정가능한 프로퍼티로 구현해."

- 기존 `CameraPanController`는 `"Horizontal"/"Vertical"` 축을 써서 WASD·방향키를 **모두** 잡았음 → 입력을 분리.
- `Assets/Scripts/CameraPanController.cs` (수정) — **방향키(↑↓←→) 전용**으로 팬 입력 변경(`Input.GetKey(KeyCode.*Arrow)`). 줌(마우스 휠)은 그대로. 플레이어는 고정, 화면만 이동(기존 방식 유지).
- `Assets/Scripts/PlayerMovement.cs` (신규) — **WASD**로 플레이어 구체를 XY 평면 이동(대각선 정규화, Z·회전 고정, Transform 이동).
  - 속도 `_moveSpeed = 3.0f` `[SerializeField]`(인스펙터 수정 가능) + 런타임 `MoveSpeed` get/set 프로퍼티(음수 방지).
- Stage 씬의 Player GameObject에 `PlayerMovement` 부착 → 저장.
- 검증: `assets-refresh` 후 컴파일 에러 0건. 실제 조작(WASD=구체 / 방향키=카메라 / 휠=줌)은 담당자가 Play로 확인.

## 테스트 방법 (Stage 씬)
1. Play → **방향키** 화면 이동(탑뷰 유지, 플레이어 고정), **WASD** 플레이어 구체 이동, **마우스 휠** 확대/축소
2. **마우스 이동** 시 빨간 레이저가 조준 방향 표시 → **좌클릭** 시 레이저 강조 + Console `발사! 방향=…, 남은 탄환=…`, 탄환 0에서 "탄환 없음"
3. GameManager 컴포넌트 우클릭 → `Debug/적 처치·민간인·플레이어 사망` 으로 클리어/실패 + 씬 전환 확인
4. (Title 씬) 좌상단 버튼 클릭으로 스테이지 시작 / 스테이지 초기화(Console `초기화`) / 다음 스테이지 / 타이틀 복귀 확인
