## 오늘 받은 요청(프롬프트)과 처리 결과

### 1) 인벤토리 구현 — 분류별 저장 + 디버그 UI
> "인벤토리를 구현해줘. 인벤토리에는 아이템, 총기 파츠, 탄환, 재화 등이 들어갈거고 후에 디버깅과 확장이 가능하도록 만들어. 같은 인벤토리에 전부 섞여 들어가는게 아니라, 각 종류별로 정렬시켜줘. UI 까지 구현해서 인벤토리에 넣었음을 확인할 수 있는 디버깅 작업까지 완료해."

설계 결정:
- **분류별 버킷**: 한 곳에 섞지 않고 `ItemCategory`(Item/GunPart/Ammo/Currency)별로 나눠 저장·표시. enum에 값만 추가하면 버킷 자동 확장.
- **데이터 ↔ UI 분리**: `Inventory`(순수 C#)와 `InventoryUI`(uGUI) 분리 → 테스트/디버그 용이.
- **스택 규칙**: `ItemDefinition.maxStack` 으로 표현. 1이면 개별 슬롯(파츠/유니크), 2 이상이면 합산(재화 등).
- **자동 부트스트랩**: `InventoryManager`(BeforeSceneLoad), `InventoryUI`(AfterSceneLoad)가 없으면 스스로 생성 → 씬 편집 없이 **어느 씬에서 Play해도** 동작(디버깅 편의).

구현(신규, `Assets/Scripts/Inventory/`):
- `ItemCategory.cs` — 대분류 enum + 한글 라벨 확장.
- `ItemDefinition.cs` — 아이템 정의 ScriptableObject(`Create > Inventory > Item Definition`). id/이름/분류/아이콘/maxStack/설명 + 런타임 생성 헬퍼.
- `InventoryEntry.cs` — 슬롯(정의+수량), 스택 Fill/Take.
- `Inventory.cs` — 분류별 버킷 저장, id 기준 스택 병합, `Add/Remove/GetQuantity/GetTotalCount/Clear` + `Changed` 이벤트.
- `InventoryManager.cs` — 전역 싱글턴(자동 부트스트랩, DontDestroyOnLoad) + 콘솔 로그.
- `InventoryUI.cs` — 우상단 uGUI 패널(자동 부트스트랩, `I` 키 토글). 디버그 추가 버튼 + 분류별 실시간 목록(`Changed` 구독).

### 2) 탄환 재설계 — 고유 강화 탄환
> "탄환은 30개씩 있는게 아니라, 기본탄환을 제외하곤 고유한 강화된 탄환이 존재할꺼야. 한 스테이지를 클리어하는데 1발에서 5발 안의 탄환으로 깨게끔 설계할꺼고, 각 탄환들은 여러 능력들이 부여되어있어서 하나하나가 고유한 탄환이 될거야."

- 초기엔 탄환을 "표준 탄환 ×30" 스택 소모품으로 뒀으나, 설계에 맞춰 **고유(비스택) 강화 탄환**으로 변경.
- `BulletItemDefinition.cs` (신규, `ItemDefinition` 상속) — 항상 Ammo 버킷 고정.
  - `isBasic=false`(강화탄) → **maxStack 1**(1발씩 개별 슬롯), `isBasic=true`(기본탄) → 스택.
  - `BulletSO bulletData` 연결 시 실제 `effects`에서 능력 목록을 가져옴(없으면 `abilityLabels`). `Create > Inventory > Bullet Item` 저작 가능.
- `InventoryUI` 디버그 교체: "탄환 ×30" 제거 → `＋ 기본 탄환` / `＋ 강화 탄환(순차)`.
  - 강화탄 샘플 4종(각 여러 능력·고유): 관통 유도탄[철갑,유도] / 폭발 분열탄[폭발,분열] / 화염 연쇄탄[화상,연쇄 번개] / 중력 냉기탄[중력,냉기].
  - 목록에서 탄환은 `· 관통 유도탄  [철갑, 유도]  ×1` 처럼 부여 능력을 함께 표시 → 하나하나 고유함을 확인.

### 4) 강화탄 ↔ 실제 BulletSO 연결 + 인벤토리 소비형 실발사
> "실제 BulletSO 효과 에셋과 강화탄 연결해줘, 일단 플레이어에게 기존 격발 코드를 넣을건데 그 밖에 더 할게 있을까?" (사용자 결정: 내가 발사 코드 구현 + 인벤토리 소비형)

- 총알 파트가 이미 저작해 둔 자산 발견: `Assets/Data/Bullets/Bullet_*.asset`(10), `Assets/Data/BulletEffects/Effect_*.asset`(9), `Assets/Prefabs/BulletPrefab.prefab`(BulletController+RB2D+Trigger). `PlayerShooter.FireBullet`는 빈 훅이었음.
- **강화탄 자산 저작**: `Assets/Resources/BulletItems/` 에 `BulletItemDefinition` 10종 생성(스크립트), 각 `bulletData`를 실제 `Bullet_*`에 연결. 강화탄=고유(maxStack 1)+한글 이름/능력 라벨, 기본탄=스택. (Resources 경유는 자동 부트스트랩 `InventoryUI`가 로드하기 위함.)
- `BulletItemDefinition.GetAbilityLabels()`: 수동 라벨 우선, 없으면 `bulletData.effects` 파생.
- `InventoryUI.BuildSamples()`: 탄환을 `Resources.LoadAll<BulletItemDefinition>("BulletItems")`로 실제 자산 로드(없으면 런타임 폴백).
- `PlayerShooter` 실발사 구현:
  - `FireBullet(BulletSO, dir)` → `Instantiate(_bulletPrefab)` + `BulletController.Init(data, dir)`.
  - `TryFire`: 인벤토리 Ammo 버킷에서 발사탄 선택(**강화탄 우선→기본탄**) → 발사 → `InventoryManager.Remove(1)` 소비. 잔탄=인벤토리 보유량(`RemainingAmmo`). 기존 `_maxAmmo/_ammo/Refill` 단순 카운터 제거.
  - `_bulletPrefab`, `_startingBullets`(시작 로드아웃) 직렬화 필드 추가. `Start()`에서 지급.
- **Stage 씬 배정(저장)**: Player의 `PlayerShooter._bulletPrefab`=BulletPrefab, `_startingBullets`=[기본 탄환, 철갑탄, 유도탄].
- 검증: `assets-refresh` 후 컴파일 에러 0건, 에셋/컴포넌트 배정값 확인. 실제 발사·충돌은 담당자 Play로 확인.

4. (Stage) ▶ Play → 좌클릭 발사: Console `[PlayerShooter] 발사! 철갑탄(Bullet_1…)`, 총알 스폰·비행, 인벤토리 Ammo 감소(강화탄 우선 소비).
