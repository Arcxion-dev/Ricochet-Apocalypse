## 오늘 받은 요청과 처리

### 1) "플레이가 안 됨" — MCP 패키지 컴파일 에러 복구
> "지금 오류 때문에 플레이가 안되…"

- 원인: 게임 코드가 아니라 **MCP 하위 패키지(ivanmurzak inputsystem/tilemap/navigation)가 끌어온 깨진 core 0.86.0** 컴파일 에러(`ProjectIdentity.DerivePortV2` 없음)가 전체 컴파일을 막음 → Play 진입 불가.
- 조치: `Packages/manifest.json` + `packages-lock.json`을 직전 커밋(core **0.84.3**)으로 되돌림 → 재해석·재컴파일 성공. **그 하위 패키지 3종은 업스트림 수정 전까지 재추가 금지.**

### 2) 적이 안 움직임 → (1차 오진) Kinematic RB
> 플레이 진입 후 적이 NavMesh 경로는 있는데 velocity=0.

- 정상 참고 씬 NavMeshPathTest의 적 RB는 **Dynamic(gravity 0)** 인데 내가 합본 적을 **Kinematic**으로 둠 → NavMeshAgent 자동 이동이 velocity=0으로 멈춘다고 판단, Dynamic으로 되돌림. (부분적으로 맞았지만 근본 원인은 아니었음 — 아래 4 참고.)

### 3) "외곽으로만 돌고 중앙 플레이어에 못 붙음" → (2차 오진) 직접 스크립트로 우회
> "적들이 외곽으로만 움직이고… 가운데에 있으면 다가오지 못해"
> "벽이 경로에 있으면 멈췄다가 부자연스럽게 떨리더니 곧 장애물을 통과…"

- NavMeshAgent가 경계에서 굳는다고 (잘못) 판단해, NavMeshAgent 없이 `NavMesh.CalculatePath`+`NavMesh.Raycast`로 직접 이동하는 `EnemyChaser2D.cs`를 만듦. → 벽에서 멈칫·떨림·통과 등 부자연스러움. **이 접근이 잘못이었음.**

### 4) 진짜 원인 발견 — Run In Background
- 디버그 로그가 재생 첫 프레임 1회만 찍히고 위치가 안 변하는 것을 보고 확인: **Unity 에디터가 포커스를 잃으면 게임 루프가 멈춘다(Player Settings의 Run In Background 꺼짐 기본).** MCP(script-execute)로 조작하며 테스트하면 적/물리가 "안 움직이는 것처럼" 보였던 것 = 실제론 게임이 일시정지.
- `Application.runInBackground=true` 켜자 즉시 정상 동작 관측. → **그동안의 "NavMeshAgent 안 움직임" 진단은 전부 이 정지 때문의 오진.**

### 5) 정식 NavMeshAgent로 확정 (골드메탈 블로그 = 표준 2D NavMesh 방식)
> "네비메쉬 컴포넌트를 활용해서 만든게 아니라, 너가 코드를 직접 짜서 추종하는 스크립트를 만든건가?" / "해당 내용 참고해서 다시 만들어봐"

- 표준 방식(검색 확인: NavMeshPlus + NavMeshSurface/CollectSources2d Bake + 적 NavMeshAgent + 코드 `agent.SetDestination(target.position)`)으로 재구성.
- Stage 적 최종 구성: **`NavMeshAgent`**(agentType 0, radius 0.4, speed 2, updateRotation/upAxis=false) + **`EnemyAIModule`**(팀원, SetDestination) + `SpeedModule` + `Enemy`(체력) + `EnemyStageBinder`(클리어 배선). **Rigidbody2D 없음**(에이전트가 transform 직접 구동). NavMeshSurface2D(NavMeshPlus, -90 회전, 베이크) 유지.
- NavMeshAgent는 navmesh에 구속 → **벽 통과·navmesh 이탈 구조적 불가**. 사용자 요구(장애물 우회 + navmesh 영역만 추격) 충족.
- 내가 만든 `EnemyChaser2D.cs`는 **삭제**.

### 6) 플레이어 2D화
> "지금 플레이어는 3d물체인데 2d로 고쳐"

- Player/Visual의 3D `MeshFilter/MeshRenderer`(구체) → 2D `SpriteRenderer`(built-in Knob, 빨강). (적 문제의 원인은 아니었음 — 콜라이더/물리 없는 순수 비주얼.)
