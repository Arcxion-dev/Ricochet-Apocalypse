# 작업일지 — 2026-07-23 (담당: KimYunjae / Player 파트) · Stage 씬 통합

Ricochet-Apocalypse · Unity 6000.3.18f1 · 2D · Legacy Input Manager · URP · NavMeshPlus(2D NavMesh)

이번 작업: **NavMeshPathTest(적 몰려옴) + BulletTest(벽 튕김)을 참고해 Stage 씬에 합쳐 실제로 작동**하게 통합.

> 확정 범위(사용자 선택): 풀 연결(총알→적 데미지 + 클리어/실패), **합본 적**(추격+체력), **대표 배치**로 간단히.

---

## 요청(프롬프트)
> "Navmeshtest 씬에서 적들이 몰려오는 것, 그리고 BulletTest에서 벽에 탄환이 튕기는 것들을 참고해서 Stage 씬에 각 오브젝트들이 제대로 연결되어 작동하도록 합쳐줘." (사전에 "작업 기록 정리 및 커밋하고 다음 작업 진행" 지시 → 탄환 스위칭 먼저 커밋)

## 조사로 확인한 구조
- **적 몰려옴(NavMeshPathTest)**: 스포너 없이 미리 배치된 `Enemy_*`가 각자 `NavMeshAgent + EnemyAIModule + SpeedModule`로 tag `Player`를 추격. 장애물은 `NavMeshModifier(area=1, NotWalkable)`로 내비메시 카빙. `NavMeshSurface2D`(NavMeshPlus `NavMeshSurface + CollectSources2d`)가 베이크.
- **벽 튕김(BulletController)**: 총알 trigger가 Wall 레이어 콜라이더에 닿으면 `Bounce`(반사). `BulletPrefab`의 `wallLayerMask=Wall(8)`/`enemyLayerMask=Enemy(7)` **이미 설정돼 있음**(07-22 메모의 "미설정"은 갱신됨).
- **레이어**: Player=6, Enemy=7, Wall=8, Bullet=9.
- **두 종류의 적이 분리**돼 있었음: `EnemyPrefab`(체력 `Enemy:Entity` O, 이동 X) vs NavMeshPathTest 씬 적(이동 O, 체력 X) → **합본 필요**.
- **데미지 미배선**: `BulletDamageDispatcher.ApplyDamage`가 로그 스텁이라 `Enemy.TakeDamage`(현재 public) 미호출.
- **NavMeshPlus 2D 수집 규칙(핵심)**: `NavMeshBuilder2d`는 **`NavMeshModifier`가 붙은 오브젝트만** 소스로 수집. `useGeometry=PhysicsColliders`면 그 오브젝트의 `Collider2D`를 사용하고, 콜라이더 위치는 **attachedRigidbody 기준**. 또 `NavMeshSurface`는 **(-90,0,0) 회전** 필요.

## 구현 내용
### 1) 배선 스크립트
- 신규 `Assets/Scripts/EnemyStageBinder.cs` (내 글루, `[RequireComponent(Enemy)]`) — 팀원 `Enemy.cs`를 건드리지 않고 클리어 판정 연결:
  - `Start`에서 `GameManager.RegisterEnemy(enemy)`, `OnDestroy`에서 처치로 간주해 `ReportEnemyKilled()` + `UnregisterEnemy()`. 씬 언로드는 `gameObject.scene.isLoaded`로 걸러 오탐 방지.
- 수정(팀원 파일, 사용자 승인) `Assets/Scripts/Bullets/BulletDamageDispatcher.cs` — 로그 스텁 → `GetComponentInParent<Entity>()?.TakeDamage(round(damage))` 실제 호출.
- 수정(팀원 파일, 사용자 승인) `Assets/Scripts/Bullets/BulletController.cs` — 민간인 피격 분기에 `GameManager.Instance?.OnCivilianHit()` 연결(실패 판정).

### 2) Stage 씬 조립 (MCP script-execute로 편집)
- **Player**: tag `Player`, layer `Player(6)`, `BoxCollider2D` 추가, 위치 (0,2). (기존 `PlayerShooter._bulletPrefab`/`_startingBullets` 배선 유지)
- **Background_Floor**: Default 레이어 큰 스프라이트(12×12, 중심 (0,2)) + `BoxCollider2D` + `Static Rigidbody2D` + `NavMeshModifier(area=0, Walkable)` → 걷는 영역 소스.
- **Obstacles**: `WallPrefab_Basic ×3` + `WallPrefab_Armored ×1`(Wall 레이어, 콜라이더+ObstacleTypeMarker 내장) 각각에 `NavMeshModifier(area=1, NotWalkable)` 추가 → 카빙 + 총알 튕김.
- **NavMeshSurface2D**: `NavMeshSurface`(AgentType 0, CollectObjects All, Size 12³, Center (0,2,0), LayerMask 257=Default+Wall, PhysicsColliders) + `CollectSources2d`, **회전 (-90,0,0)**.
- **적(합본) ×4** `Enemy_Chaser_0..3`: `EnemyPrefab`(체력100) 인스턴스에 `NavMeshAgent`(radius 0.4, speed 2) + `SpeedModule`(base 2) + `EnemyAIModule` + `EnemyStageBinder` 추가, `Rigidbody2D`는 `Kinematic`(에이전트 구동), 네 모서리 배치.
- **베이크**: `NavMeshSurface.BuildNavMesh()` 후 `Assets/Scenes/Stage_NavMesh.asset`로 영속화(씬 저장 시 참조 유지).

## 검증 (플레이 모드 제어 툴 미가용 → 정적 검증)
- 베이크 로그: `Sources 5`, navmesh **정점 102 / 삼각형 44**(바닥 walkable, 벽 카빙).
- 경로 계산: `NavMesh.SamplePosition`로 Player·적4 모두 onNavMesh=True, `NavMesh.CalculatePath` **4/4 적이 플레이어까지 PathComplete**.
- 스크린샷(Front, 톱뷰): 중앙 Player, 네 모서리 적4, 벽3+장갑벽1, 걷는 바닥 확인.
- 컴파일/콘솔 Error·Exception 0건.

## 흐름 (런타임)
1. 적4가 NavMesh로 벽을 피해 Player 추격(EnemyAIModule 0.3s마다 목적지 갱신).
2. 총알이 벽(Wall)에 맞으면 튕김(최대 `BulletSO.maxBounceCount`), 적(Enemy)에 맞으면 `BulletDamageDispatcher`→`Enemy.TakeDamage` 데미지.
3. 적 체력 0 → `Enemy` 파괴 → `EnemyStageBinder.OnDestroy` → `ReportEnemyKilled`+`UnregisterEnemy` → 마지막 적 사망 시 `GameManager.StageClear` → 상점 씬.

## ⚠️ 비고 / 남은 협의
- **팀원 파일 2개**(`BulletDamageDispatcher.cs`, `BulletController.cs`)를 수정함 — 사용자 승인 하에 진행. 총알 파트와 최종 공유 필요.
- **밸런스**: `EnemyPrefab` 체력 100 vs `BulletSO.damage`가 작으면 1~5발 컨셉과 안 맞음(수치 조정은 밸런싱 몫). 배선/작동은 검증 완료.
- 적끼리/적-플레이어 물리 밀림 없음(적 Kinematic + 에이전트 회피). 적 공격 로직은 아직 없음(요청 범위 밖 — "몰려옴"까지).
- 대표 배치라 벽/적 수·위치는 임시. 정식 레벨 배치는 맵 파트와 협의.
