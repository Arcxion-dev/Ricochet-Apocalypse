## 오늘 받은 요청(프롬프트)과 처리 결과

### 1) 탄환 스위칭 + 좌하단 탄환 전용 UI
> "인벤토리에서 원하는 탄환을 발사할 수 있도록 스위칭 기능 추가 1,2,3,4,5로 선택 가능. 화면 왼쪽아래에 탄환 인벤토리만 UI 따로 표시."

설계 결정: **선택 상태의 주인은 `PlayerShooter`**. HUD는 그걸 읽어 표시만.

- `PlayerShooter` (수정):
  - 인벤토리 Ammo 버킷을 **종류별로 묶어** 최대 5종 선택 목록(`Choices`) 구성(같은 id/기본탄 스택은 수량 합산). `Inventory.Changed` 구독으로 발사·획득 시 자동 갱신(`RebuildChoices`).
  - **숫자키 1~5**(상단+키패드) → `SelectBullet(index)`로 발사할 탄 선택.
  - `TryFire`가 기존 "강화탄 우선 자동선택" 대신 **선택된 탄**을 발사(`ResolveSelectedBullet`, 선택 목록 비면 기존 규칙으로 폴백).
  - HUD 연동용 공개 API: `Choices`, `SelectedIndex`, `SelectionChanged` 이벤트, `MaxSelectableBullets=5`.
- `BulletInventoryUI.cs` (신규, `Assets/Scripts/UI/`):
  - **화면 좌하단** 탄환 전용 HUD. 각 줄 `[번호] 이름 ×수량 [능력]`, **선택 탄은 ▶ + 노란 굵게** 강조.
  - 런타임 코드 생성(`RuntimeInitializeOnLoadMethod`) → 씬 편집 불필요. 표시 전용(버튼 없음)이라 EventSystem/Raycaster 불필요.
  - `PlayerShooter` 있는 씬에서만 표시, 없으면 자동 숨김. 선택/목록 변경은 가벼운 시그니처 비교로 감지해 갱신. 우상단 전체 `InventoryUI`와 별개.

### 2) 인벤토리 열림 시 조준-호흡-격발 입력 잠금
> "인벤토리가 열려있을 때는 조준-호흡-격발이 입력이 안되게 해줘."

- `InventoryUI.IsOpen`(static) 추가 — 우상단 인벤토리 패널 표시 여부 노출.
- `PlayerShooter.Update`: `InventoryUI.IsOpen`이면 조준-호흡-격발 로직 스킵(좌클릭 조준고정/우클릭취소/격발 모두 차단, 레이저도 정지). 탄환 선택(1~5)은 계속 동작.

### 3) 시작 시 인벤토리 닫힘
> "시작시 인벤토리가 닫혀있게."

- `InventoryUI._startVisible` 기본값 `true`→`false`. 시작하자마자 사격 가능, `I` 키로 열면 그동안 사격 차단.

## 테스트 방법 (Stage 씬 Play)
1. 시작 시 우상단 인벤토리 닫힘, 좌하단 탄환 HUD 표시(시작 로드아웃 [기본, 철갑, 유도]).
2. 숫자키 1~5로 발사할 탄 선택 → HUD에서 ▶ 강조 이동, Console `[PlayerShooter] 탄환 선택 → [n] …`.
3. 조준→호흡→격발로 **선택한 탄**이 나가고 인벤토리에서 1발 소비 → HUD 수량 갱신.
4. `I`로 인벤토리 열면 조준/발사 입력 차단(레이저 정지), 닫으면 재개.
